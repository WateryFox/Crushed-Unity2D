I developed this project to explore the concept of vertical level design and environmental hazards in a 2D space. Unlike a traditional platformer where you move toward a finish line, this game is about survival and constant downward movement. The player must navigate between platforms while avoiding a variety of deadly obstacles, such as spikes and a bottomless void. The most unique mechanic I implemented is the "crushing" threat, which forces the player to stay mobile and react quickly to the changing landscape. My goal was to create a game loop that feels tense and rewarding by balancing risk and reward in a confined, high pressure environment.

The technical core of the game is the hazard system, which I built using Unity’s 2D Trigger and Collision components. I had to write specific logic to differentiate between a simple platform landing and a "crush" event. To achieve this, I used raycasting and multiple collision points on the player character to detect if they were being squeezed between two moving objects or a platform and a boundary. The "void" and "spikes" were implemented as kill zones that immediately trigger a game over state, which helped me practice efficient scene management and UI transitions. Managing these different death conditions required a clean "Health" or "Life" script that could handle various damage sources without overlapping.

I focused heavily on making the downward movement feel smooth and responsive. I used a C# script to control the player's gravity and air friction, ensuring that the fall speed felt consistent and predictable. One of the biggest challenges was making sure the platforms felt solid while still allowing for fast paced movement. I also worked on a simple UI system that tracks how long the player has survived or how many platforms they have successfully descended. This project taught me how to use the Unity Camera system to follow the player vertically while maintaining a fixed horizontal view, which is essential for "infinite" style survival games.
